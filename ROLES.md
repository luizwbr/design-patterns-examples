## Role Transition Process

### Becoming a Contributor
1. Fork the repository
2. Make your changes following [CONTRIBUTING.md](CONTRIBUTING.md)
3. Submit a pull request with DCO sign-off
4. Once merged, you're a contributor!

### Becoming a Collaborator
1. Contribute regularly over several months
2. Demonstrate quality and understanding of project standards
3. Show commitment to helping the community
4. Receive invitation from Project Lead
5. Accept collaborator invitation on GitHub

### Stepping Down
Any role holder may step down by:
1. Notifying the Project Lead (or community if you are the Lead)
2. Completing any in-progress critical tasks if possible
3. Transferring knowledge or responsibilities

**For comprehensive succession and continuity planning**, see [ACCESS_CONTINUITY.md](ACCESS_CONTINUITY.md).
